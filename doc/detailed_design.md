## 2. オフライン計算アルゴリズム

### 2.1 日柱の算出 (本質コードの基準)
1. **基準（アンカー）点**: `1980年3月10日` (協定世界時 UTC基準) = 干支番号 **19番**。
2. **経過日数 ($Days$)**:
   $$Days = \text{round}\left( \frac{\text{Date.UTC}(Y, M-1, D) - \text{Date.UTC}(1980, 2, 10)}{86,400,000} \right)$$
3. **干支番号 ($Idx$)**:
   $$Idx = ((18 + Days) \bmod 60 + 60) \bmod 60 + 1$$
4. **日干インデックス ($StemIdx$)**:
   $$StemIdx = (Idx - 1) \bmod 10$$
   *(0:甲, 1:乙, 2:丙, 3:丁, 4:戊, 5:己, 6:庚, 7:辛, 8:壬, 9:癸)*

### 2.2 二十四節気（節入り日）の略算式 (月柱・年柱判定用)
四柱推命の月・年の境界線を正確に判定するための略算式です。

$$Day_{term} = \lfloor D + A \times (Y' - 1900) - \lfloor \frac{Y' - 1900}{4} \rfloor \rfloor$$
* $m \in \{1, 2\}$ の場合: $Y' = Year - 1$
* $m \ge 3$ の場合: $Y' = Year$
* $D, A$ 定数は基本設計書で定義された各月の定数を使用。

### 2.3 十二運星（動物）の算出ロジック
日干インデックス ($StemIdx$) と、判定対象の地支インデックス ($BranchIdx$) から十二運星を決定します。

* **陽干（$StemIdx$ が偶数: 甲, 丙, 戊, 庚, 壬） ➔ 順行**:
  $$StageIdx = (BranchIdx - Start_{yang} + 12) \bmod 12$$
  * $Start_{yang}$ マップ: `{ 0(甲): 11, 2(丙): 2, 4(戊): 2, 6(庚): 5, 8(壬): 8 }`

* **陰干（$StemIdx$ が奇数: 乙, 丁, 己, 辛, 癸） ➔ 逆行**:
  $$StageIdx = (Start_{yin} - BranchIdx + 12) \bmod 12$$
  * $Start_{yin}$ マップ: `{ 1(乙): 6, 3(丁): 9, 5(己): 9, 7(辛): 0, 9(癸): 3 }`

* **動物キャラクターへの変換**:
  得られた $StageIdx$ (0〜11) は、次の順序で配列から引かれます。
  `['猿', 'チーター', '黒ひょう', 'ライオン', '虎', 'たぬき', 'コアラ', 'ゾウ', 'ひつじ', 'ペガサス', '狼', 'こじか']`

## 3. オンライン・スクレイピング方式仕様 (ラッパー版 API)

### 3.1 外部リクエスト仕様
* **リクエスト先**: `https://www.doubutsu-uranai.com/uranai_chara_5animals.php`
* **通信メソッド**: `POST`
* **ヘッダー**:
  * `Content-Type`: `application/x-www-form-urlencoded`
  * `User-Agent`: `Mozilla/5.0 ...` (ボット検知回避のため、主要ブラウザのユーザーエージェントをシミュレート)
* **送信パラメータ**:

| キー名 | 値の形式 | 目的 |
| :--- | :--- | :--- |
| `nickname` | 文字列 (固定値: `ゲスト`) | 必須入力フォームの充足 |
| `gender` | `M` (男性) または `F` (女性) | 性別判定（男女で結果は変わりませんが必須） |
| `birth_array[]` (1つ目) | 西暦4桁 (例: `1986`) | 生年 |
| `birth_array[]` (2つ目) | 2桁の月 (例: `04`) | 生月 |
| `birth_array[]` (3つ目) | 2桁の日 (例: `26`) | 生日 |
| `method` | 固定値: `input` | フォーム送信判定値 |
| `submit` | 固定値: `占う` | 送信アクションシグナル |

### 3.2 HTMLパース＆データ抽出仕様
応答HTMLテキストに対し、以下のグローバル正規表現を実行します。
```javascript
const regex = /alt="([^"]+キャラ：[^"]+)"/g;
```
* **マッチング例**: `alt="表面キャラ：こじか"`
* **データ分割**: `：`（全角コロン）で分割し、前半を「役割」、後半を「動物名」として認識。
* **役割のパース条件**:
  * `本質` を含む ➔ `essential` (本質)
  * `表面` を含む ➔ `surface` (表面)
  * `意思決定` を含む ➔ `decision` (意思決定)
  * `希望` を含む ➔ `hope` (希望)
  * `隠れ` を含む ➔ `hidden` (隠れ)
* **JSON応答モデル**:
  ```json
  {
    "essential": "024",
    "surface": "108",
    "decision": "000",
    "hope": "125",
    "hidden": "012"
  }
  ```

## 4. フロントエンドUI詳細仕様

### 4.1 ダイヤモンド（十文字）グリッドレイアウト
ラッパー版の5枚のスタンプは、CSS Gridの `3x3` のグリッドセルに配置されます。

```css
.stamps-grid {
  display: grid;
  grid-template-columns: repeat(3, 112px);
  grid-template-rows: repeat(3, auto);
  gap: 16px 20px;
}
```

* 表面 (2nd) ➔ `grid-column: 2; grid-row: 1;`
* 希望 ➔ `grid-column: 1; grid-row: 2;`
* 本質 (1st) ➔ `grid-column: 2; grid-row: 2;`
* 意思決定 (3rd) ➔ `grid-column: 3; grid-row: 2;`
* 隠れ ➔ `grid-column: 2; grid-row: 3;`

### 4.2 アニメーション遅延 (スパイラル刻印)
スタンプ表示クラス `.reveal` に対するアニメーション遅延設定です。

```css
.essential-box .reveal { animation-delay: 0s; }      /* 1. 本質 */
.surface-box .reveal   { animation-delay: 0.12s; }   /* 2. 表面 */
.decision-box .reveal  { animation-delay: 0.24s; }   /* 3. 意思決定 */
.hope-box .reveal      { animation-delay: 0.36s; }   /* 4. 希望 */
.hidden-box .reveal    { animation-delay: 0.48s; }   /* 5. 隠れ */
```

### 4.3 入力検証 (バリデーション)
* **生年月日フォーマット**: 半角数字8桁。全角数字は `normalize` 関数で自動的に半角に置換。
* **実在日判定**: 
  JavaScriptの `Date` は無効な日付（例: 2月30日）を入力すると自動的に翌月に繰り越すため、`Date.UTC(y, m-1, d)` で変換した後の値が、入力された `y, m, d` と一致するかをチェックして無効な日付を弾きます。
* **入力制限範囲**:
  * オフライン版: `1900〜2050年` (節入り日計算の精度維持のため)
  * ラッパー版: `1800〜2026年` (公式サイト側の入力フォームの選択範囲に基づく)
